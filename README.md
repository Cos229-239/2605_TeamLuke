
## Branch Workflow

This repository follows the class branch workflow:

1. `main` is the stable branch.
2. `dev` is the shared team development branch.
3. Personal work happens on a personal branch, such as `LukePersonal`.
4. Personal branches should pull/merge from `dev` before new work.
5. Work should be committed and pushed to the personal branch before any team merge into `dev`.
